🏷️ Placeholder

Fornece uma dica visual sobre o que deve ser inserido no campo

<input type="text" placeholder="Digite seu nome completo">
💡 Dica: O placeholder desaparece quando o usuário começa a digitar e não substitui o label.

🎯 Autofocus

Foca automaticamente no campo quando a página carrega

<input type="email" autofocus placeholder="seu@email.com">
⚠️ Atenção: Use apenas um autofocus por página para não confundir o usuário.

📏 Size

Define a largura visual do campo em caracteres

<input type="text" size="4" maxlength="4" placeholder="0000">
📝 Nota: O size define apenas a largura visual, não limita a quantidade de caracteres.

📐 Maxlength

Limita o número máximo de caracteres que podem ser digitados

<input type="text" maxlength="15" placeholder="usuario123">
✅ Benefício: Evita que o usuário digite mais do que o permitido, melhorando a UX.

⚠️ Required

Torna o preenchimento do campo obrigatório

<input type="text" required placeholder="Digite seu nome">
🔒 Validação: O navegador impede o envio do formulário se campos required estiverem vazios.

📝 Lista de Descrição

As listas de descrição são usadas para criar pares de termos e suas definições, ideais para glossários, FAQs e dicionários.

Elementos da Lista de Descrição:

<dl> - Define a lista de descrição (Description List)

<dt> - Define o termo a ser descrito (Description Term)

<dd> - Define a descrição do termo (Description Details)

<dl class="glossario"> <dt>HTML</dt> <dd>HyperText Markup Language - Linguagem de marcação para criar páginas web</dd> <dt>CSS</dt> <dd>Cascading Style Sheets - Linguagem para estilizar páginas web</dd> <dt>JavaScript</dt> <dd>Linguagem de programação para adicionar interatividade às páginas web</dd> </dl>

💬 Citações

As tags de citação são usadas para marcar conteúdo citado de outras fontes, melhorando a semântica e acessibilidade.

Elementos de Citação:

<blockquote> - Para citações longas em bloco, geralmente com quebra de linha

<q> - Para citações curtas inline, automaticamente adiciona aspas

<cite> - Para referenciar o título de uma obra (livro, filme, artigo)

<blockquote cite="https://developer.mozilla.org"> <p>A web é para todos, e coletivamente temos a responsabilidade de mantê-la acessível.</p> <footer>— <cite>Mozilla Developer Network</cite></footer> </blockquote> <p>Como disse Tim Berners-Lee: <q>A web foi projetada para ser descentralizada.</q></p> <p>Recomendo a leitura do livro <cite>Don't Make Me Think</cite> de Steve Krug.</p>

Exemplo Prático:

A web é fundamentalmente projetada para funcionar para todas as pessoas, independentemente de seu hardware, software, idioma, localização ou capacidade.

— Web Accessibility Initiative (WAI)

Steve Jobs uma vez disse: Design não é apenas como algo parece e como se sente. Design é como funciona.

Para aprender mais sobre UX, recomendo o livro The Design of Everyday Things de Don Norman.

🏠 Tag Main

A tag <main> define o conteúdo principal de um documento, excluindo cabeçalhos, rodapés, barras laterais e navegação.

Características da Tag Main:

Único por página: Deve haver apenas um elemento <main> por documento

Conteúdo principal: Contém o conteúdo central e único da página

Acessibilidade: Ajuda leitores de tela a identificar o conteúdo principal

SEO: Melhora a compreensão do conteúdo pelos mecanismos de busca

<!DOCTYPE html> <html> <head> <title>Minha Página</title> </head> <body> <header> <nav>Menu de navegação</nav> </header> <main> <h1>Conteúdo Principal</h1> <article> <p>Este é o conteúdo principal da página...</p> </article> </main> <aside>Barra lateral</aside> <footer>Rodapé</footer> </body> </html>

Boas Práticas:

• Não inclua elementos repetidos como navegação, cabeçalho ou rodapé dentro do <main>

• Use apenas um <main> por página

• Coloque o conteúdo mais importante e único da página dentro do <main>

• Combine com outras tags semânticas como <article>, <section> e <aside>

📍 Tag Address

A tag <address> fornece informações de contato para o autor ou proprietário do documento ou seção.

Uso da Tag Address:

Informações de contato: Email, telefone, endereço físico, redes sociais

Escopo: Pode ser para toda a página ou para uma seção específica

Não é para endereços gerais: Apenas para contato do autor/proprietário

Estilização padrão: Geralmente renderizada em itálico pelos navegadores

<!-- Informações de contato do autor da página --> <address> <p>Escrito por <a href="mailto:joao@exemplo.com">João Silva</a></p> <p>Telefone: <a href="tel:+5511999999999">(11) 99999-9999</a></p> <p>Endereço: Rua das Flores, 123 - São Paulo, SP</p> </address> <!-- Informações de contato dentro de um artigo --> <article> <h2>Como Criar Sites Acessíveis</h2> <p>Conteúdo do artigo...</p> <address> <p>Por <a href="mailto:maria@webdev.com">Maria Santos</a></p> <p>Desenvolvedora Front-end na WebDev Company</p> </address> </article>

🖼️ Tag Figure

A tag <figure> representa conteúdo independente, como imagens, diagramas, códigos, com uma legenda opcional usando <figcaption>.

Elementos Figure:

<figure> - Container para conteúdo independente

<figcaption> - Legenda ou descrição do conteúdo

Conteúdo: Imagens, gráficos, códigos, tabelas, vídeos

Independente: Pode ser movido sem afetar o fluxo do documento

<figure> <img src="grafico-vendas.png" alt="Gráfico mostrando aumento de 30% nas vendas"> <figcaption>Figura 1: Crescimento das vendas no último trimestre</figcaption> </figure> <figure> <pre><code> function calcularMedia(numeros) { const soma = numeros.reduce((a, b) => a + b, 0); return soma / numeros.length; } </code></pre> <figcaption>Código 1: Função para calcular média de um array</figcaption> </figure>
